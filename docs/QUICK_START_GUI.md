# GUI 快速开始指南

## 5分钟快速上手

### 第一步：安装依赖

```bash
# 进入项目目录
cd smart-scheduling

# 安装所有依赖
pip install -r requirements.txt
```

### 第二步：启动GUI

```bash
streamlit run gui_app.py
```

启动成功后，浏览器会自动打开，或访问显示的URL（通常是 `http://localhost:8501`）。

### 第三步：运行第一个调度任务

#### 3.1 加载示例订单

1. 点击顶部的 **📦 订单管理** 标签页
2. 点击右侧的 **📂 加载小规模示例** 按钮
3. 看到订单概览显示约20个订单

#### 3.2 运行调度

1. 点击顶部的 **🚀 调度运行** 标签页
2. 设置 **模拟天数** 为 `3`
3. 点击 **▶️ 运行调度** 按钮
4. 等待几秒钟，算法运行完成

#### 3.3 查看结果

1. 点击顶部的 **📊 结果分析** 标签页
2. 查看生成的甘特图和各项指标
3. 可以下载图表保存

## 常用操作

### 调整算法参数

**场景**：想要更快的运行速度

1. 在侧边栏将 **种群规模** 改为 `20`
2. 将 **最大迭代次数** 改为 `30`
3. 点击 **💾 应用配置**
4. 重新运行调度

**场景**：想要更好的优化结果

1. 在侧边栏将 **种群规模** 改为 `50`
2. 将 **最大迭代次数** 改为 `100`
3. 点击 **💾 应用配置**
4. 重新运行调度

### 使用自定义订单

1. 准备CSV文件，格式如下：
```csv
order_id,product,quantity,due_slot,unit_price
1,2,220,5,70
2,1,150,6,50
```

2. 在 **📦 订单管理** 标签页上传CSV
3. 查看订单概览确认数据正确
4. 运行调度

### 导出配置

1. 在 **📋 配置详情** 标签页
2. 点击 **📥 导出配置(JSON)**
3. 保存配置文件供后续使用

### 下载结果图表

1. 在 **📊 结果分析** 标签页
2. 点击对应的下载按钮：
   - 下载甘特图
   - 下载利润图
   - 下载订单图
   - 下载利用率图

## 结果解读

### 财务指标

- **总收入**：完成订单获得的收入
- **总成本**：人工成本（不同时段成本不同）
- **总罚款**：未完成订单的违约罚款
- **总利润**：收入 - 成本 - 罚款

**优化目标**：最大化总利润

### 订单指标

- **完成率**：完全完成的订单占比
- 理想值：>90%
- 如果过低，考虑：
  - 增加模拟天数
  - 调整订单截止时间
  - 优化算法参数

### 甘特图解读

- **横轴**：时间段（每段4小时）
- **纵轴**：生产线（Line 1-3）
- **颜色块**：不同产品的生产任务
- **标注**：订单ID和生产数量

**查看要点**：
- 产线是否充分利用
- 是否有大量空闲时段
- 订单是否在截止时间前完成

## 常见问题

### Q：GUI启动失败？

**A**：检查是否安装了streamlit
```bash
pip install streamlit
```

### Q：调度运行报错？

**A**：可能原因：
1. 订单数据格式不正确
2. 参数设置不合理
3. 检查控制台错误信息

### Q：结果图表不显示？

**A**：
1. 确保调度已成功运行
2. 检查 `output/` 目录是否有PNG文件
3. 刷新页面重试

### Q：如何停止GUI？

**A**：在终端按 `Ctrl+C`
