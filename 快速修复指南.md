# 快速修复指南：生产线空闲问题

## 问题
使用 `delay.csv` 进行5天调度时，生产线大量空闲。

## 原因
**订单量不足！** 只有7,150单位，而5天产能是14,850单位，利用率仅48.1%。

## 解决方法（3选1）

### ✅ 方法1：使用满负荷数据（推荐）

直接使用已生成的满负荷订单数据：

```
文件位置: data/delay_full.csv
订单数量: 48个
产能利用率: 90.0%
```

**在GUI中使用:**
1. 打开GUI应用
2. 在"订单管理"标签页
3. 上传 `data/delay_full.csv` 文件
4. 运行调度

### 🔧 方法2：生成自定义数据

如果需要不同的产能利用率：

```bash
# 编辑 generate_full_orders.py
# 修改第15行: target_utilization = 0.90  # 改为你想要的值 (0.7-0.95)

python generate_full_orders.py
```

生成的文件会保存到 `data/delay_full.csv`

### 📊 方法3：保持现状（测试轻负荷场景）

如果你就是想测试轻负荷场景（48%利用率），那么当前的空闲是**正常现象**，不需要修复。

## 常见问题

**Q: 为什么不是算法问题？**
A: 因为订单总量(7,150) < 总产能(14,850)，物理上不可能填满。

**Q: 如何判断数据是否合理？**
A: 产能利用率应在80-95%之间。低于60%说明订单严重不足。

**Q: 可以修改产能配置吗？**
A: 可以，但不推荐。更好的方法是调整订单量匹配产能。
